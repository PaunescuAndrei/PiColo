1.Introducere

Aplicatie de tip service worker pentru optimizarea imaginilor de pe pagini web.

Aplicatia are urmatoarele caracteristici:

  • Utilizatorul va putea gestiuna o multitudine de service workers pentru diferite domenii.
  • Utilizatorul va putea seta diferiti parametrii pentru service worker-ul de pe un anumit domeniu.
  • Service worker-ul va optimiza imaginile de pe pagini web folosing parametrii setati de utilizator.
  • Acesta va trimite inapoi imaginile optimizate.

2.Tehnologii

Tehnologiile folosite in creearea acestei aplicatii sunt afisate mai jos.

2.1. HTML

HyperText Markup Language (HTML) este un limbaj de marcare utilizat pentru crearea paginilor web ce pot fi afisate intr-un browser (sau navigator). Scopul HTML este mai degraba prezentarea informatiilor – paragrafe, fonturi, tabele s.a.m.d. – decat descrierea semanticii documentului.

2.2. CSS

CSS (Cascading Style Sheets) este un standard pentru formatarea elementelor unui document HTML. Stilurile se pot atașa elementelor HTML prin intermediul unor fisiere externe sau direct in cadrul documentului html.

2.3. JavaScript

JavaScript (JS) este un limbaj de programare orientat obiect bazat pe conceptul prototipurilor. Este folosit mai ales pentru introducerea unor funcționalitati în paginile web, codul JavaScript din aceste pagini fiind rulat de catre browser.

2.4. GitHub

GitHub este un serviciu de gazduire web pentru proiecte de dezvoltare a software-ului care utilizeaza sistemul de control al versiunilor Git.

3.Arhitectura aplicatiei

Structura generala a aplicatiei

3.1. Structura generala a serviciului

Service worker-ul intercepteaza toate requesturile pentru imagini si trimite un raspuns cu imaginile procesate de catre backend.

Service worker-ul functioneaza astfel:

    ー Acesta intercepteaza toate requesturile pentru imagini.
    ー Acesta trimite requesturile la backend cu parametrii specificati.
    ー Backend-ul verifica daca requesturile sunt deja procesate.
    ー In cazul in care imaginile au fost deja procesate se va trimite imagine din cache.
    ー In caz contrar, incep procesarea imaginii.

4.Mod de utilizare

Mai jos este descris un scenariu normal de utilizare a aplicatiei.

4.1. Gestionare

Utilizatorul va putea sa acceseze o pagina unde va putea sa gestioneaze diversi service workeri pentru diferite domenii.

4.2. Setare parametrii

Utilizatorul va putea seta diversi parametrii privind procesarea imaginilor.

    ー Specificare tipului de compresie (lossy/lossless).
    ー Specificarea watermark-ului pe imaginile procesate.
    ー Configurarea printr-o expresie regulata URL-urile imaginilor.

4.3. Adaugarea worker-ului.

Utilizatorul va adauga service worker-ul generat in paginile in care doreste sa recompreseze imaginile.
